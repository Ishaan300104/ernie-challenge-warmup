Web Builder using PaddleOCR & ERNIE

Automated PDF-to-Web Pipeline for the ERNIE Challenge Warm-up Task

Objective

The goal of this warm-up task is to demonstrate an end-to-end pipeline that converts PDF documents into a deployable web page using OCR, document understanding, and large language models. This project showcases the integration of PaddleOCR-VL for optical character recognition and layout analysis, combined with ERNIE's powerful language generation capabilities to create production-ready web content.

Overall Workflow

The complete pipeline follows a systematic approach to transform static PDF documents into dynamic, accessible web pages. Each stage builds upon the previous one, creating a seamless conversion process.

1
Input PDF Document
A PDF document is provided as input. This document may contain structured text, headings, tables, and layout information. The system accepts various PDF formats and preserves the original document structure throughout the conversion process.
2
Text and Layout Extraction using PaddleOCR-VL
PaddleOCR-VL is used to extract textual content along with layout-level information such as paragraphs, headings, lists, and structural blocks from the PDF pages. The advanced vision-language model understands document semantics and preserves hierarchical relationships. The output of this stage is a structured representation of the document content with maintained formatting and layout.
3
Conversion to Markdown
The extracted text and layout information are converted into clean, well-structured Markdown format. Headings, bullet points, code blocks, and sections are preserved to maintain document readability and hierarchy. This intermediate format provides flexibility for further processing and ensures consistent structure across different document types.
4
Web Page Generation using ERNIE Model
The Markdown content is passed to the ERNIE language model. ERNIE is used to generate a complete HTML-based web page from the Markdown input. This includes semantic structuring, section organization, and content refinement suitable for a web interface. The model applies intelligent styling decisions and ensures optimal user experience through responsive design and accessibility features.
5
Static Website Deployment
The generated HTML content is prepared as a static website. The website is deployed using GitHub Pages, enabling public access through a hosted URL. The deployment process is automated through CI/CD pipelines, ensuring rapid iteration and updates. The final webpage is optimized for performance, SEO, and cross-browser compatibility.

Dataset Context

The dataset used for this workflow has been web-scraped and curated from Arduino official documentation and Arduino user forums. This ensures technical accuracy and real-world relevance for the generated content. The curated dataset provides diverse examples of technical documentation, making it ideal for testing OCR accuracy and content understanding capabilities.

Key Features

  • Automated Processing: End-to-end pipeline requiring minimal human intervention
  • Layout Preservation: Maintains document structure including headings, lists, and hierarchies
  • Intelligent Generation: ERNIE model applies semantic understanding to create coherent web content
  • Responsive Design: Generated pages adapt seamlessly to different screen sizes
  • Deployment Ready: Direct integration with GitHub Pages for instant publishing
  • Accessibility: Semantic HTML ensures content is accessible to all users

Outcome

The final output is a fully functional static web page generated automatically from a PDF document, demonstrating the seamless integration of OCR, document understanding, language modeling, and deployment technologies. This project showcases how modern AI technologies can work together to transform traditional documents into modern, accessible web content.

This warm-up task serves as a foundation for more advanced applications in document processing, knowledge extraction, and automated content generation. The pipeline can be extended to handle various document types, languages, and output formats, making it a versatile solution for digital transformation initiatives.
